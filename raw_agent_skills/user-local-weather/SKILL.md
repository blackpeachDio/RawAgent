# 用户本地天气（先地址后天气）

## 何时使用

- 用户用「我这儿」「这里」「本地」「我们这边」等指代当前所在地，并询问天气、气温、下雨等。
- **且**用户没有直接说出城市名（如「北京天气」则不必按本流程先查地址）。

## 工具调用顺序（必须遵守）

1. **确定用于查地址的姓名**  
   - 若用户自称或对话里能确定其姓名（与侧栏「当前用户名」或上下文中的称呼一致），用该姓名作为 `get_user_location` 的入参。  
   - 若只有用户名/昵称而无真实姓名映射，仍用当前对话中可用的**用户标识字符串**尝试；若工具返回「未知城市」，则**不要编造城市**，应简要说明无法从资料中解析城市，并请用户说出城市名后再调用 `get_weather`。

2. **查城市**  
   - 调用工具 **`get_user_location(user_name)`**，得到城市名字符串（如「深圳」）。  
   - 若结果为「未知城市」或空，**不要**继续调用 `get_weather`；改为请用户补充城市。

3. **查天气**  
   - 用上一步得到的**确切城市名**调用 **`get_weather(city)`**。  
   - 将返回的字符串整理后回答用户；不要复述工具名或中间 JSON。

## 注意

- `get_user_location` 依赖应用内演示映射（姓名 → 城市）；生产环境若未配置真实用户档案，应引导用户直接说城市。  
- 天气数据来自公网 `wttr.in`，需网络可达；失败时如实告知用户。
