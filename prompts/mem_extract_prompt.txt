你是用户记忆提取专家。
从对话中提取：事实、事件。

【对话内容】
以下为最近若干轮用户与助手的对话，按时间从旧到新排列；请综合全部轮次判断，不要只盯着最后一轮。

{conversation}

【输出格式】
仅输出一段合法 JSON 文本：第一行以左花括号开始，最后一行以右花括号结束。
禁止 Markdown 代码围栏、说明文字、注释等；除 JSON 外不要输出任何字符。
JSON 结构中只包含两个 key："facts" 和 "events"，其 value 都是列表。

结构示例（请严格按此结构输出，不要输出本行说明）：
{{
    "facts": [
        {{"key": "xxx", "value": "xxx"}},
        {{"key": "hobby", "value": "绘画", "scenario": "业余时间"}},
        {{"key": "xxx", "value": "xxx"}}
    ],
    "events": [
        "事件描述1",
        "事件描述2"
    ]
}}

【规则】
1. facts 只能是用户固定属性：
   允许的 key：name, age, job, city, hobby, character, preferences, avoid, rules
2. 对 hobby、character、preferences：同一 key 可在多轮对话中多次出现；每次只写**本轮新出现或更新**的条目（系统会按日期合并，与旧条目并存）。
   可选字段 scenario：简短说明适用场景（如「工作」「周末」「饮食」），便于后续按场景检索；无则省略。
3. name/age/job/city/avoid/rules 等单值字段仍用单条 key+value 覆盖语义即可。
4. events 是发生的事情、对话、经历、动作（可来自任意一轮）
5. 跨轮信息（如前几轮提到的居住地、偏好）若仍有效，应写入 facts 或 events
6. 没有则填空数组
